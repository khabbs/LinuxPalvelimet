g)SSH-demonin asennus


23.15 ajoin seuraavat komennot:
 ‘sudo apt-get install openssh-server’ #OpenSSH asennus
‘sudo systemctl enable ssh’ #SSH service enablointi
‘sudo systemctl start ssh’ #SSH service käynnistys
Loin SSH avaimen ilman salasanaa (huom tarkoituksella root, testaan onko root kirjautuminen defaultina sallittu)
‘sudo ssh-keygen’ #SSH avaimen luonti
‘ssh -i /usr/bin/mykey user@host’ #

Git
Klo 23.45 Asensin sovelluksen komennolla ‘sudo apt install git’
Avasin SSH avaimen komennolla ‘cat /usr/bin/mykey.pub’ ja lisäsin sen GitHubiin lähdesivuston ⁶ ohjeella.
Loin uuden paikallisen git repon ‘git init’ ja vein tämän raportin sinne ‘git add /home/jj/Downloads/h2.txt’
Lisäsin Githubin osoitteen, jonne repository lähetetään ‘git remote add origin https://github.com/khabbs/hh’
Koitin lisätä repository muutokset Github repooni ‘git push origin master’, mutta sain virheilmoituksen:
error: src refspec master does not match any.
error: failed to push some refs to 'https://github.com/khabbs/hh'
Unohdin tallentaa muutokset ja valmistella ne uploadia varten. Ajoin komennon ‘git commit -m "TEST"’ ja sen jälkeen uudelleen komennon ‘git push origin master’. Nyt muutokset eli uusi kansio ja tiedosto, näkyvät GitHubissa https://github.com/khabbs/hh/blob/master/Downloads/h2.txt
